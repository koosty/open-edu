## Enrollment Management

### Viewing Enrolled Students

**Access Student List**:
1. Go to **Admin Panel** → **Your Course**
2. Click **"Students"** or **"Enrollments"** tab
3. See complete list of enrolled students

**Student Information Displayed**:
- Student name
- Profile picture
- Enrollment date
- Current progress percentage
- Last activity date
- Number of lessons completed

### Enrollment Methods

**Self-Enrollment** (Default):
- Students enroll themselves
- No approval needed
- Instant access to course
- Most common method

**Instructor Approval**:
1. Student requests enrollment
2. You receive notification
3. Review request
4. **Approve** or **Deny** enrollment
5. Student notified of decision

**Manual Enrollment**:
1. Go to student list
2. Click **"Add Student"**
3. Enter student email
4. Student receives notification
5. Instant enrollment

### Unenrolling Students

**When to Unenroll**:
- Student drops the course
- Incorrect enrollment
- Course completion
- Student request

**How to Unenroll**:
1. Find student in list
2. Click **"Unenroll"** or **"Remove"**
3. Confirm action
4. Student loses access immediately
5. Progress data preserved

> **Note**: Unenrolling does NOT delete student's work or data. It only removes access to the course.

## Monitoring Student Progress

### Progress Overview

**Course-Level Metrics**:
- Overall completion percentage
- Number of lessons completed
- Quiz scores
- Time spent in course
- Last accessed date

**Accessing Individual Progress**:
1. Go to student list
2. Click student name
3. View detailed progress page

### Student Progress Page

**What You'll See**:

**1. Progress Summary**
- Overall progress bar
- Lessons completed (e.g., "12/20 lessons")
- Average quiz score
- Total time spent
- Completion status

**2. Lesson-by-Lesson Progress**
```
Lesson 1: Introduction ✓ Completed • 100% • 15 min
Lesson 2: Variables ✓ Completed • 100% • 20 min
Lesson 3: Functions → In Progress • 45% • 8 min
Lesson 4: Objects ○ Not Started • 0%
```

**3. Quiz Performance**
- Quiz name
- Attempts made
- Scores per attempt
- Pass/fail status
- Time taken

**4. Activity Timeline**
- Recent lessons accessed
- Quiz attempts
- Completion milestones
- Login history

### Progress Indicators

**Completion Status**:
- ✓ **Completed**: Green checkmark
- → **In Progress**: Arrow icon
- ○ **Not Started**: Empty circle
- ⚠ **At Risk**: Warning indicator

**Reading Progress**:
- Percentage of lesson scrolled/read
- Updated in real-time
- Indicates engagement level

**Quiz Status**:
- **Passed**: Green badge
- **Failed**: Red badge
- **In Progress**: Blue badge
- **Not Attempted**: Gray

## Identifying At-Risk Students

### Warning Signs

**Academic Risk Indicators**:
- ⚠️ Falling behind schedule
- ⚠️ Multiple failed quiz attempts
- ⚠️ Low quiz scores (<60%)
- ⚠️ Incomplete lessons
- ⚠️ Not progressing for 7+ days

**Engagement Risk Indicators**:
- ⚠️ Infrequent logins (once per week or less)
- ⚠️ Short session times (<5 minutes)
- ⚠️ Skipping lessons
- ⚠️ Not taking notes
- ⚠️ Abandoning quizzes mid-way

### At-Risk Dashboard

**Access the Dashboard**:
1. Go to **Course → Analytics**
2. Click **"Students"** tab
3. Toggle **"Show At-Risk Only"** filter

**At-Risk Criteria** (automatic):
- Progress <25% after 2 weeks
- Failed quiz attempts >3
- No activity in last 7 days
- Quiz average <60%
- Low engagement score

**Dashboard Features**:
- Sort by risk level
- Filter by specific issues
- Bulk actions available
- Export list for outreach

### Intervention Strategies

**Early Intervention** (Week 1-2):
- Send welcome message
- Check if students can access content
- Offer office hours
- Share study tips

**Mid-Course Check-In** (Week 3-5):
- Email students falling behind
- Offer tutoring or extra resources
- Review quiz performance
- Address specific struggles

**Late Intervention** (Week 6+):
- One-on-one meetings
- Extension requests
- Referral to academic support
- Discuss completion plans

### Proactive Support

**Automated Notifications** (if available):
- Congratulate on milestones
- Reminder for inactive students
- Quiz retry encouragement
- Course completion reminders

**Manual Outreach**:
- Personal email or message
- Group announcements
- Office hours invitations
- Peer study groups

## Communication Strategies

### Announcement System

**Creating Announcements**:
1. Go to **Course → Announcements**
2. Click **"New Announcement"**
3. Write message (supports markdown)
4. Choose recipients:
   - All students
   - Specific students
   - At-risk students only
5. Click **"Post Announcement"**

**Best Practices**:
- Keep announcements concise
- Use clear subject lines
- Post regularly but not excessively
- Highlight important deadlines
- Include action items

**Announcement Types**:
- Welcome message (start of course)
- Weekly updates
- Deadline reminders
- New content notifications
- Encouragement and motivation

### Direct Messaging

**Messaging Individual Students**:
1. Go to student's profile
2. Click **"Send Message"** or **"Email"**
3. Write personal message
4. Send

**When to Message**:
- Student requests help
- Notice concerning pattern
- Congratulate achievement
- Clarify confusion
- Provide personalized feedback

**Message Templates**:

**Falling Behind**:
```
Hi [Name],

I noticed you haven't accessed the course in a week. 
Is everything okay? Let me know if you need help 
catching up or if circumstances have changed.

Best,
[Your Name]
```

**Congratulations**:
```
Hi [Name],

Great work passing the quiz with 95%! Your dedication 
is paying off. Keep up the excellent progress!

Best,
[Your Name]
```

**Quiz Struggles**:
```
Hi [Name],

I see you're having trouble with the Functions quiz. 
This is a challenging topic! Would you like to discuss 
the concepts during office hours? I'm here to help.

Best,
[Your Name]
```

### Office Hours & Support

**Setting Up Office Hours**:
- Announce time and method (video call, chat, etc.)
- Use scheduling tool (Calendly, Google Calendar)
- Post link in course announcement
- Remind students weekly

**Best Practices**:
- Offer multiple time slots
- Record sessions (with permission)
- Share recordings with all students
- Prepare FAQ based on common questions

## Grading and Feedback

### Automated Grading

**Auto-Graded Question Types** (5 types):
- Multiple Choice
- Multiple Answer
- True/False
- Short Answer
- Fill in the Blank

**What Happens**:
- Instant grading upon submission
- Score calculated automatically
- Results shown immediately
- No instructor action needed

### Manual Grading

**Essay Questions**:
- Require instructor review
- Student submits essay
- You receive notification
- Grade and provide feedback
- Student sees updated score

**Grading Essay Questions**:
1. Go to **Course → Quizzes → [Quiz Name]**
2. Click **"Pending Grades"** tab
3. See list of essays to grade
4. Click student submission
5. Read essay response
6. Assign points (0 to max points)
7. Write feedback comments
8. Click **"Submit Grade"**
9. Student notified of score

**Grading Rubric Example**:
```
Question: Explain closures in JavaScript (5 points)

Rubric:
- Definition (2 points)
- Code example (2 points)
- Explanation of scope (1 point)

Feedback Template:
"Your definition was clear (2/2), and your example 
demonstrated understanding (2/2). Consider adding 
more detail about lexical scope (0.5/1). 

Total: 4.5/5 - Excellent work!"
```

### Providing Constructive Feedback

**Effective Feedback**:
- ✅ Specific: Point to exact issues
- ✅ Actionable: Suggest improvements
- ✅ Balanced: Note strengths and weaknesses
- ✅ Timely: Grade within 24-48 hours
- ✅ Encouraging: Motivate improvement

**Feedback Examples**:

❌ **Poor**: "Good job"
✅ **Better**: "Your explanation of functions was clear, especially the example with parameters. Next time, add more detail about return values."

❌ **Poor**: "Wrong"
✅ **Better**: "You're close! Review the difference between == and ===. Your code example needs strict equality for this use case."

### Grade Adjustment

**When to Adjust**:
- Technical issue during quiz
- Ambiguous question
- Incorrect answer key
- Student appeals grade

**How to Adjust**:
1. Go to student's quiz attempt
2. Click **"Adjust Grade"** or **"Override"**
3. Enter new score
4. Provide reason
5. Save changes
6. Student notified

> **Important**: Document all grade adjustments for accountability.

## Managing Course Access

### Access Permissions

**By Default, Enrolled Students Can**:
- View all published lessons
- Take quizzes (if available)
- Track their progress
- Add notes and bookmarks
- Download course materials (if enabled)

**Instructors Can Control**:
- Lesson availability
- Quiz access
- Content downloads
- Discussion forums (if available)

### Lesson Prerequisites

**Setting Prerequisites** (if feature available):
1. Go to lesson settings
2. Enable **"Require Prerequisites"**
3. Select prerequisite lesson(s)
4. Save settings

**How It Works**:
- Student must complete prerequisite first
- Lesson is locked until prerequisite done
- Progress bar shows locked lessons
- Useful for sequential learning

### Quiz Access Control

**Quiz Availability Options**:

**1. Always Available**
- Students can take anytime
- No restrictions
- Best for practice quizzes

**2. Date Range**
- Available from start date to end date
- Example: "Available March 1-7"
- Auto-closes after end date

**3. After Specific Lesson**
- Unlocks after lesson completion
- Enforces sequential learning

**4. Manual Release**
- Instructor manually opens quiz
- Useful for in-class assessments

### Extending Deadlines

**Individual Extensions**:
1. Go to student's profile
2. Find quiz or assignment
3. Click **"Extend Deadline"**
4. Set new deadline
5. Student notified

**Common Extension Scenarios**:
- Medical emergencies
- Technical difficulties
- Academic accommodations
- Personal circumstances

**Best Practices**:
- Require documentation for extensions
- Be consistent with policies
- Communicate extensions clearly
- Document all extensions

## Student Data & Privacy

### What Data is Collected

**Progress Data**:
- Lessons completed
- Quiz scores
- Time spent
- Last activity

**Activity Data**:
- Login times
- Page views
- Notes created
- Bookmarks saved

**Personal Data**:
- Name
- Email
- Profile picture
- Enrollment date

### Privacy Considerations

**Instructor Can See**:
- ✅ All course activity
- ✅ Quiz attempts and scores
- ✅ Progress metrics
- ✅ Student notes (if configured)

**Instructor Cannot See**:
- ❌ Password or login credentials
- ❌ Activity in other courses
- ❌ Personal messages (unless sent to you)
- ❌ System-level data

**Best Practices**:
- Only view data for legitimate educational purposes
- Don't share student data publicly
- Follow institutional privacy policies (FERPA, GDPR)
- Secure your instructor account
