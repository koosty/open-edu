# Video Tutorial Script: Creating a Quiz with QuizBuilder

**Target Audience**: Instructors  
**Duration**: 8-10 minutes  
**Goal**: Demonstrate how to create a comprehensive quiz with various question types using QuizBuilder

---

## Introduction (30 seconds)

**On Screen**: Open-Edu admin dashboard

**Voiceover**:
"Welcome, instructors! In this tutorial, you'll learn how to create engaging quizzes using Open-Edu's QuizBuilder. We'll cover all question types, quiz settings, and best practices. Let's dive in!"

---

## Part 1: Accessing QuizBuilder (1 minute)

**On Screen**: Admin dashboard navigation

**Voiceover**:
"From the admin dashboard, navigate to the Courses section."

**On Screen**: 
1. Click "Courses" in sidebar
2. Show list of courses
3. Click "Manage" on a course card

**Voiceover**:
"Select the course where you want to add a quiz. Click 'Manage' to open the course editor."

**On Screen**: 
4. Course editor showing lesson list
5. Click "Add New" button
6. Select "Quiz" from lesson type dropdown

**Voiceover**:
"In the course editor, click 'Add New' and select 'Quiz' from the lesson type dropdown. This opens the QuizBuilder interface."

---

## Part 2: Quiz Basic Settings (1.5 minutes)

**On Screen**: QuizBuilder interface - Quiz Settings tab

**Voiceover**:
"Let's start by configuring the basic quiz settings. The QuizBuilder has two main tabs: Quiz Settings and Questions."

**On Screen**: Focus on Quiz Title field

**Voiceover**:
"First, give your quiz a descriptive title. I'll call this 'Module 1 Assessment: HTML Basics.'"

**On Screen**: Type title, then scroll to Time Limit field

**Voiceover**:
"Next, set a time limit in minutes. For this 10-question quiz, I'll set 20 minutes. This gives students about 2 minutes per question."

**On Screen**: Enter "20" in time limit field

**Voiceover**:
"The time limit should be reasonableâ€”consider question complexity and whether students need to reference materials."

**On Screen**: Scroll to Passing Score field

**Voiceover**:
"Set a passing score as a percentage. I'll set 70%, which is standard for most quizzes."

**On Screen**: Enter "70" in passing score field

**Voiceover**:
"Students scoring 70% or higher will pass and can download a certificate. You can adjust this based on quiz difficulty."

**On Screen**: Scroll to Max Attempts field

**Voiceover**:
"Decide how many times students can retake the quiz. I'll allow 3 attempts. This lets students learn from mistakes without unlimited retakes."

**On Screen**: Enter "3" in max attempts field

## Part 3: Adding Questions Overview (30 seconds)

**On Screen**: Click "Questions" tab

**Voiceover**:
"Now let's add questions. Click the Questions tab to access the question builder."

**On Screen**: Show empty questions list with "Add Question" button

**Voiceover**:
"You can create six types of questions: Multiple Choice, Multiple Answer, True/False, Fill in the Blank, Short Answer, and Ordering. Let's add one of each type."

## Part 4: Multiple Choice Questions (1 minute)

**On Screen**: Click "Add Question" button

**Voiceover**:
"Click 'Add Question' to create your first question."

**On Screen**: Question Type dropdown showing "Multiple Choice" (default)

**Voiceover**:
"Multiple Choice is selected by default. This is best for questions with one correct answer."

**On Screen**: Type in Question Text field

**Voiceover**:
"Enter your question. I'll ask: 'What does HTML stand for?'"

**On Screen**: Show four answer options

**Voiceover**:
"Multiple choice requires 2-6 answer options. Let's add four options."

**On Screen**: 
1. Type Option A: "Hyper Text Markup Language"
2. Type Option B: "High Tech Modern Language"
3. Type Option C: "Home Tool Markup Language"
4. Type Option D: "Hyperlinks and Text Markup Language"

**Voiceover**:
"Make sure incorrect options are plausible but clearly wrong to students who know the material."

**On Screen**: Select option A as correct (click radio button)

**Voiceover**:
"Mark the correct answer by selecting its radio button. Option A is correct."

**On Screen**: Click "Save Question"

---

## Part 5: Multiple Answer Questions (1 minute)

**Voiceover**:

3. Type Option C: "<span>" - check correct
4. Type Option D: "<section>" - check correct

"Check all correct answers. Students must select ALL correct options to get pointsâ€”partial credit isn't given."

**Voiceover**:

**Voiceover**:
"Select the correct answer. This statement is true, so I'll select True."

**On Screen**: Click "Save Question"

**Voiceover**:
"True/False questions are graded automatically and are quick for students to answer."

## Part 7: Fill in the Blank Questions (1 minute)

**On Screen**: Add new question, select "Fill in the Blank"

**Voiceover**:
"Fill in the Blank tests recall and specific knowledge."

**On Screen**: Type question: "The _____ tag is used to create a hyperlink in HTML."

**Voiceover**:
"Use underscores or brackets to indicate where students should fill in the answer."

**Voiceover**:
"Here's a pro tip: provide multiple acceptable answers to account for variations."

**On Screen**: 
1. Type "<a>" in first line
2. Type "a" in second line
3. Type "anchor" in third line

**Voiceover**:
"Students can answer '<a>', 'a', or 'anchor'â€”all are correct. This reduces frustration from minor variations."

**Voiceover**:
"Remember: Fill in the Blank matching is case-insensitive and trims whitespace automatically."

## Part 8: Short Answer Questions (1 minute)

**On Screen**: Add new question, select "Short Answer"

**Voiceover**:
"Short Answer questions require written responses and must be manually graded."

**On Screen**: Type question: "Explain the difference between inline and block-level elements in HTML."

**Voiceover**:
"Use Short Answer for questions requiring explanation, analysis, or critical thinking."

**On Screen**: Show "Sample Answer" field (optional)

**Voiceover**:
"You can optionally provide a sample answer as a grading guide for yourself or other instructors."

**Voiceover**:
"This helps maintain consistent grading standards, especially with multiple instructors."

**On Screen**: Click "Save Question"

## Post-Production Notes

### Visual Highlights
- Zoom in when showing specific fields or buttons
- Highlight important settings with colored boxes
- Add arrows pointing to key interface elements

### Editing Tips
- Speed up repetitive typing slightly
- Add keyboard sound effects when typing
- Include transition effects between major sections
- Add subtle "save" confirmation animations

### On-Screen Text Overlays
- Question type names when introducing each type
- Key tips (e.g., "Allow multiple correct answers for Fill in the Blank")
- Best practice summaries at the end

### Accessibility
- Provide closed captions for all voiceover
- Ensure UI elements are clearly visible (high contrast)
- Use readable fonts for text overlays (16pt minimum)
- Include descriptive audio for visual actions
